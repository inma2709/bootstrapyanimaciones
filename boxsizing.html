Posicionamiento en CSS: Grid (contenedores) + Flex (elementos)

Aprende a alinear, distribuir y reordenar contenedores con Grid y elementos con Flex, y a dominar la unidad fr (p. ej. 3fr 1fr).

Índice de la lección

  1. Conceptos clave: ejes, “justify-” vs “align-”, hijos vs contenedor.
  2. Flexbox: posicionar y reordenar elementos dentro de un contenedor.
  3. Grid: alinear ítems y mover el grid completo; áreas; self.
  4. Unidad fr en Grid (qué significa 3fr 1fr y cómo usarlo bien).
  5. Playgrounds para practicar (Flex y Grid) + ejercicios propuestos.

1) Conceptos clave

Terminología esencial

  • Contenedor: el elemento padre que activa display:flex o display:grid.
  • Ítem: los hijos directos del contenedor.
  • justify-*: actúa sobre el eje principal (en LTR, horizontal por defecto).
  • align-*: actúa sobre el eje cruzado (en LTR, vertical por defecto).
  • order (Flex): reordena visualmente sin cambiar el HTML.
  • grid-area / grid-column / grid-row (Grid): colocan ítems en celdas concretas.

Reglas de oro

  • Flex = posicionar y distribuir elementos en línea o columna.
  • Grid = definir estructura bidimensional (columnas + filas).
  • justify-self existe en Grid (por ítem), no en Flex.
  • Reordenar visualmente no cambia el orden del DOM (accesibilidad).

Piensa: “Grid para el layout macro; Flex para organizar dentro de cada tarjeta/sección”.

2) Posicionamiento con Flexbox (contenedor & elementos)

Contenedor Flex — propiedades principales:

  • flex-direction: row | row-reverse | column | column-reverse
  • justify-content (eje principal): flex-start | center | flex-end | space-between | space-around | space-evenly
  • align-items (eje cruzado): flex-start | center | flex-end | stretch | baseline
  • gap: separación entre ítems.

Ítems Flex (hijos) — propiedades útiles:

  • order: entero (admite negativos).
  • align-self: sobreescribe align-items para ese ítem.
  • margin-left:auto (fila) o margin-top:auto (columna): “empujan” el ítem al extremo.

4) La unidad fr en Grid (qué es y cómo se reparte)

fr significa “parte fraccional del espacio disponible”. El contenedor reparte el espacio libre entre pistas (columnas/filas) que usan fr.

  • 1fr 1fr → 50% / 50% (dos partes iguales).
  • 2fr 1fr → 66% / 33% (dos partes para la 1ª, una para la 2ª).
  • 3fr 1fr → 75% / 25% (tres partes vs una). OJO: no es “3,1fr”. Se escriben dos pistas: grid-template-columns: 3fr 1fr;
  • Si mezclas valores fijos y fr, primero se restan los fijos y el resto se reparte en fr.
Regla mental: suma los “fr” y reparte el ancho libre en esa proporción.

Más combinaciones útiles con fr (copia y prueba) 
/* Tres columnas: 2/1/1 → 50% / 25% / 25% aprox. */
grid-template-columns: 2fr 1fr 1fr;

/* Mezcla con fijos: primera 300px fija, resto al 50/50 */
grid-template-columns: 300px 1fr 1fr;

/* Resiliente: minmax con fr (crece pero no baja de 240px) */
grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));

Crea tantas columnas como quepan en el contenedor, cada una de al menos 240px, que pueden expandirse hasta repartirse el ancho sobrante (1fr cada una).

Configuración de columnas en CSS Grid

Sí, puedes definir las columnas como quieras (2fr 1fr 3fr, por ejemplo). Pero ojo: las unidades fr reparten el espacio libre del contenedor y respetan el tamaño mínimo del contenido. Eso significa que si en una columna hay un texto muy largo o algo “inrompible”, esa columna puede crecer más de lo esperado (rompiendo la proporción).

Reglas rápidas

  • fr = fracción del espacio libre (tras restar gap y columnas fijas).
  • El contenido impone límites mínimos (min-content).
  • Para que las fracciones se cumplan mejor y evitar desbordes, usa minmax(0, …).

Ejemplo básico

.grid {
  display: grid;
  grid-template-columns: 2fr 1fr 3fr; /* 2:1:3 del espacio libre */
  gap: 1rem;
}

Variante recomendada

.grid {
  display: grid;
  grid-template-columns: minmax(0, 2fr) minmax(0, 1fr) minmax(0, 3fr);
  gap: 1rem;
}

Consejos útiles

  • Si quieres que una columna nunca sea más pequeña que su contenido: minmax(min-content, 2fr).
  • Para layouts fluidos de tarjetas: grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));.
  • Si hay palabras muy largas: word-break: break-word; o overflow-wrap: anywhere; en los ítems.

¿Qué significa grid-template-columns: repeat(auto-fit, minmax(240px, 1fr))?

  1. repeat(...): evita escribir varias veces lo mismo (repeat(3, 1fr) = 1fr 1fr 1fr).
  2. auto-fit: mete tantas columnas como quepan; si sobran huecos, se colapsan.
  3. minmax(240px, 1fr): cada columna mide al menos 240px y puede crecer hasta ocupar 1fr.

En conjunto, significa: “Crea tantas columnas como quepan en el contenedor, cada una de mínimo 240px y máximo 1fr”.

Ejemplo práctico

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 1rem;
}

<div class="grid">
  <div class="card">A</div>
  <div class="card">B</div>
  <div class="card">C</div>
  <div class="card">D</div>
</div>

🔎 Resultado:

  • En pantallas anchas, aparecen varias columnas de igual ancho.
  • Al reducir la ventana, las tarjetas se recolocan hasta apilarse en 1 sola columna cuando no caben 240px.

6) Ejercicios rápidos

  1. Crea un Grid de 3 columnas y 6 tarjetas. En móvil, 1 columna (repeat(auto-fit,minmax(280px,1fr))).
  2. Dentro de una tarjeta, pon botón al extremo derecho con margin-left:auto (fila).
  3. Define un layout con grid-template-areas y mueve “nav” arriba cambiando su grid-area por “header”.
  4. Usa 3fr 1fr para un layout “principal + sidebar”. Cambia a 2fr 1fr y compara.
Recuerda: usa Grid para el esqueleto de página y Flex para ordenar contenido dentro de cada bloque. Practica cambiando valores inline: se entiende “con las manos”.
