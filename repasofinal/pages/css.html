1) Selectores básicos

Tipo (etiqueta)

Afecta a todas las etiquetas del mismo tipo. 

p { line-height: 1.6; }
h2 { color: var(--color-secundario); }
p
Este <p> obtiene el interlineado.

Clase (.clase)

Reutilizable en muchos elementos. 

.destacado { background: #fff8ff; color: var(--color-base); }
.destacado
Bloque con clase "destacado".

ID (#id)

Único por documento. Es muy específico. 

#intro { border-left: 4px solid var(--color-base); padding-left: .5rem; }
#intro
Soy el bloque con id "intro".

Atributo

Muy útil con formularios y enlaces.

input[type="email"] { border-color: var(--color-base); }
a[target="_blank"] { text-decoration-style: dotted; }
input[type="email"]
<input type="email"> recibe estilo distinto.

4) Cascada y especificidad (¿quién gana?)

Orden en la cascada

A igualdad de especificidad, gana la regla que aparece más tarde.

p { color: #333 }      /* 1º */
p { color: #111 }      /* 2º  ← GANA */
La segunda regla pisa la primera si apuntan a lo mismo.

Especificidad

Jerarquía: !important > inline style > #id > .clase/[atributo]/:pseudo-clase > tipo/::pseudo-elemento. 

p { color: blue }          /* tipo */
.highlight p { color: red }/* clase + tipo → GANA */
#box p { color: green }    /* id + tipo → GANA a la anterior */
#box y .highlight superan al selector de tipo.

¡Cuidado con !important!

Úsalo como último recurso; dificulta el mantenimiento. 

.boton { color:#222 !important }
nav .boton { color:#999 }  /* NO pisa al anterior por !important */
!important gana incluso con menos especificidad.

Tip de depuración

En DevTools, mira “Computed” para ver la regla ganadora y su especificidad. Evita selectoritis: prefiere selectores claros y poco profundos.

  • Prefiere .btn--primario a header nav ul li a.btn.
  • Centraliza colores con :root y var().

5) Herencia (qué se transmite)

Propiedades que heredan

Ej.: color, font-family, line-height.

.panel { color:#222; font-family:var(--fuente-principal); }
.panel p em { /* heredan color y fuente */ }
Soy el padre → los hijos heredan color y fuente.

Propiedades que NO heredan

Ej.: margin, padding, border, background.

.panel { padding:16px; }
.panel p { /* NO hereda padding */ }
El padding es del contenedor, no de los párrafos dentro.

Forzar herencia / reinicios

inherit, initial, unset.

.hijo { color: inherit; }     /* toma el del padre */
.reset { all: initial; }      /* valores iniciales de navegador */
.clean { all: unset; }        /* hereda si es heredable; si no, inicial */
Padre azul → hijo hereda azul

Mini guía práctica

  • Define tipografías/colores en body o contenedores altos → el resto hereda.
  • Reinicia con cuidado: all: unset borra mucho más de lo que imaginas.
body{ color:#333; line-height:1.6; }
h1,h2{ color:var(--color-secundario); }

6) Resumen rápido

Resumen

  • Selectores: elige el correcto (simple, combinador, pseudo).
  • Cascada: a igual especificidad, gana el que va último.
  • Especificidad: #id > .clase/[attr]/:pseudo > etiqueta/::pseudo.
  • Herencia: tipografía/colores sí; cajas (margin/padding/border), no.

Cheat-sheet de especificidad

/* Cuenta puntos (aprox) */
#id            → 100
.clase [attr] :pseudo  → 010
etiqueta ::pseudo      → 001
inline style           → 1000
!important             → máximo (evítalo)

Cuando dudes, simplifica el selector y revisa en DevTools cuál gana.

7) Test de conocimientos

Pregunta 1 - Tipos de selectores

¿Cuáles son los tipos básicos de selectores en CSS?

Ver respuesta

Respuesta: Selector de etiqueta (ej: p), de clase (ej: .clase), de id (ej: #id) y de atributo (ej: input[type="text"]).

Pregunta 2 - Personalizar Bootstrap

¿Cómo podemos añadir nuestro propio estilo a una web que usa Bootstrap?

Ver respuesta

Respuesta: Añadiendo una hoja de estilos propia después de la de Bootstrap en el HTML. Así, nuestros estilos sobrescriben los de Bootstrap si tienen igual o mayor especificidad.

Pregunta 3 - Especificidad

¿Cuál de estos selectores tiene mayor especificidad?
A) #header .nav
B) nav.principal
C) body nav ul li

Ver respuesta

Respuesta A: #header .nav (110 puntos: 1 ID + 1 clase) tiene mayor especificidad que B (020 puntos: 2 clases) y C (004 puntos: 4 elementos).

Pregunta 4 - Grid Templates

¿Qué hace grid-template-columns: 1fr 2fr 1fr?

Ver respuesta

Respuesta: Crea 3 columnas donde la del medio (2fr) ocupa el doble de espacio que las laterales (1fr cada una). Es como dividir el espacio en 4 partes: 1+2+1.

Pregunta 5 - Herencia

Si aplico color: red a un div, ¿heredarán ese color los párrafos dentro?

Ver respuesta

Respuesta: SÍ. La propiedad color es heredable, por lo que todos los elementos hijos (incluyendo párrafos) heredarán el color rojo, a menos que tengan un color específico definido.

Pregunta 6 - Combinadores

¿Qué selecciona h2 + p?

Ver respuesta

Respuesta: Selecciona el primer párrafo <p> que aparece INMEDIATAMENTE después de un <h2>, siendo hermanos al mismo nivel. Solo el primer párrafo adyacente, no todos.

Pregunta 7 - Grid Areas

¿Para qué sirve grid-template-areas?

Ver respuesta

Respuesta: Permite definir áreas nombradas en el grid de forma visual y legible, como "header header" "nav main" "footer footer". Luego usas grid-area: header para posicionar elementos.

Pregunta 8 - Pseudo-clases

¿Cuál es la diferencia entre :nth-child(2n) y :nth-child(even)?

Ver respuesta

Respuesta: No hay diferencia. Ambos seleccionan los elementos en posiciones pares (2, 4, 6, 8...). even es simplemente una forma más legible de escribir 2n.

Pregunta 9 - Grid vs Flexbox

¿Cuándo usar Grid y cuándo Flexbox?

Ver respuesta

Respuesta: Grid: para layouts 2D (filas Y columnas), como páginas completas. Flexbox: para layouts 1D (una dirección), como barras de navegación, botones o centrar contenido.

Pregunta 10 - !important

¿Cuándo es apropiado usar !important?

Ver respuesta

Respuesta: Casi nunca. Solo en casos muy específicos como: sobrescribir estilos de librerías externas, clases utilitarias (ej: .hidden { display: none !important; }) o correcciones urgentes temporales.

8) Flexbox y Grid Templates

Flexbox: Layout en 1 dimensión

Perfecto para: Barras de navegación, centrar elementos, distribuir espacio entre items.

Propiedades del contenedor:

.contenedor {
  display: flex;
  flex-direction: row | column;
  justify-content: center | space-between | space-around;
  align-items: center | flex-start | flex-end;
  gap: 1rem;
}

Propiedades de los items:

.item {
  flex: 1;        /* crece para ocupar espacio */
  flex-shrink: 0; /* no se encoge */
  align-self: center; /* alineación individual */
}

Grid: Layout en 2 dimensiones

Perfecto para: Layouts de página completa, galerías, formularios complejos.

Definir la cuadrícula:

.grid-container {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr; /* 3 columnas */
  grid-template-rows: auto 1fr auto;  /* 3 filas */
  gap: 1rem;
  height: 100vh;
}

Posicionar elementos:

.header { 
  grid-column: 1 / -1; /* de la primera a la última columna */
}
.sidebar { 
  grid-row: 2 / 3; /* fila 2 */
}

Grid Template Areas: Layout Visual

La forma más intuitiva de crear layouts complejos:

.layout {
  display: grid;
  grid-template-areas: 
    "header  header  header"
    "nav     main    aside"
    "footer  footer  footer";
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: auto 1fr auto;
  gap: 1rem;
  min-height: 100vh;
}

.header { grid-area: header; }
.nav    { grid-area: nav; }
.main   { grid-area: main; }
.aside  { grid-area: aside; }
.footer { grid-area: footer; }

Ventaja: Es como dibujar el layout con texto. Muy legible y fácil de modificar.

¿Cuándo usar cada uno?

Usa Flexbox cuando:

  • Necesitas alinear elementos en UNA dirección
  • Quieres centrar contenido fácilmente
  • Los elementos deben crecer/encogerse dinámicamente
  • Ejemplo: navbar, botones, cards en fila

Usa Grid cuando:

  • Necesitas control sobre filas Y columnas
  • Quieres posicionar elementos en áreas específicas
  • El layout es complejo (header, sidebar, main, footer)
  • Ejemplo: página completa, galería de fotos, dashboard

Tip: ¡Puedes combinarlos! Un grid para el layout general y flexbox para los componentes internos.

Grid Responsivo Fácil

Usando repeat() y minmax():

/* Grid que se adapta automáticamente */
.grid-auto {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1rem;
}

/* Significa: crea todas las columnas que quepan,
   cada una mínimo 250px y máximo el espacio restante */

Con esta técnica, el grid se reorganiza automáticamente según el tamaño de pantalla. ¡Sin media queries!

Patrones Flexbox Comunes

/* Centrar perfecto */
.centro {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Navbar típico */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* Cards que crecen igual */
.cards {
  display: flex;
  gap: 1rem;
}
.card { flex: 1; }

/* Sticky footer */
.page {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}
.main { flex: 1; }
