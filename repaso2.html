Repaso: Estructura + Grid Areas

Corrige el HTML roto, añade semántica y aplica tu Grid

1) Código HTML “roto” (para corregir)

Objetivo: Arregla la estructura y añade semántica. Luego maqueta con el Grid que ves más abajo.

  • Hay texto dentro del <head>.
  • No se abre la etiqueta <body>.
  • La etiqueta <html> no está cerrada.
  • No hay encabezados (<h1>–<h6>).
  • Todo son <div> y el “cuerpo” son cajas sin Grid.
<!doctype html>
<html lang="es">
<head>
  ESTO ES UN TEXTO QUE NO DEBERÍA ESTAR AQUÍ
  <meta charset="utf-8">
  <title>Ejercicio de repaso</title>
</head>

<div class="caja">Caja 1</div>
<div class="caja">Caja 2</div>
<div class="caja">Caja 3</div>
<div class="caja">Caja 4</div>
<div class="caja">Caja 5</div>
<div class="caja">Caja 6</div>
<div id="grid">Aquí no hay Grid, solo divs</div>

Tu objetivo mínimo

  • Corrige al menos 3 errores de estructura (texto en <head>, abrir <body>, cerrar </html>…).
  • Incluye al menos 7 etiquetas semánticas de: <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>, <h1>, <h2>, <p>, <ul>, <li>, <figure>, <img>.

Maquetación (usa este Grid)

Aplica esta plantilla de áreas en tu CSS (ya incluida arriba):

.grid-container {
  display: grid;
  grid-template-areas:
    "header header header"
    "banner main aside"
    "footer footer footer";
  grid-template-columns: 150px 1fr 220px;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}
header { grid-area: header; }
.banner { grid-area: banner; }
main   { grid-area: main; }
aside  { grid-area: aside; }
footer { grid-area: footer; }

Idea: en escritorio verás 3 columnas (banner / main / aside). En móvil, el media query lo apila en una columna.

Sugerencia: crea tu propio archivo estilos.css y enlázalo con:
<link rel="stylesheet" href="estilos.css">. Así separas estructura (HTML) de presentación (CSS).

3) ¿Qué se ha corregido y qué aporta?

  • Texto fuera de lugar en <head> → eliminado: el <head> solo lleva metadatos (mejor validez e interoperabilidad).
  • <body> → añadido como contenedor de todo el contenido visible.
  • </html> → cerrado para completar la estructura del documento.
  • Semántica incorporada (≥7 etiquetas): <header>, <nav> (como banner), <main>, <article>, <section>, <aside>, <footer>, además de <h1>–<h3>, <p>, <figure>, <figcaption>, <ul>, <li>.
  • Grid con áreas (tu solución): .grid-container define el mapa con grid-template-areas y asignamos cada región con grid-area. Esto da un layout claro (3 columnas en desktop), y con media query se apila en móvil.
  • Accesibilidad: landmarks claros para lectores de pantalla (header/nav/main/aside/footer) y jerarquía de encabezados correcta.
  • Mantenibilidad: separar HTML de CSS permite iterar estilos sin tocar la estructura.

Reto extra: crea tu estilos.css con tu paleta y variantes (hover, spacing, tipografías) y enlázalo con:
<link rel="stylesheet" href="estilos.css">.
