Aprendiendo CSS

CSS (Cascading Style Sheets) es el lenguaje fundamental para estilizar y dar presentación a tus páginas web. Mientras HTML se encarga de la estructura del contenido, CSS le da vida, permitiéndote controlar colores, fuentes, tamaños, diseños y mucho más. Es la clave para transformar una página básica en una experiencia visual atractiva y profesional.

En esta sección, nos sumergiremos en los conceptos esenciales de CSS, desde la selección de elementos y la aplicación de estilos básicos hasta técnicas avanzadas de diseño responsivo. Descubrirás cómo crear diseños fluidos, animaciones sutiles y asegurar que tus sitios web luzcan impecables en cualquier dispositivo.

🌱 Introducción a CSS

¡Bienvenido al mundo de CSS! Si HTML es el esqueleto de tu página web, CSS (Cascading Style Sheets) es la piel, el maquillaje y la ropa. Es el lenguaje que usaremos para darle estilo, color y vida a cada uno de los elementos de tu sitio.

¿Qué es CSS y para qué sirve?

CSS es un lenguaje de hojas de estilo que se utiliza para describir la presentación de un documento escrito en un lenguaje de marcado como HTML. Su principal propósito es separar el contenido de una página de su diseño. Esto significa que puedes cambiar el aspecto visual de un sitio web completo modificando un solo archivo CSS.

Sirve para todo lo relacionado con la apariencia: desde el color y el tamaño de las fuentes, hasta la disposición de los elementos en la pantalla. Gracias a CSS, podemos crear diseños que se adaptan a diferentes dispositivos, aplicar efectos de transición o animaciones, y mejorar la experiencia de usuario.

Diferencia entre HTML (estructura) y CSS (presentación)

HTML (Estructura)

  • Función: Define el esqueleto y el contenido.
  • Pregunta clave: "¿Qué hay en la página?"
  • Ejemplos: Un <p> es un párrafo, un <h1> es un título principal.
  • Metáfora: La estructura de una casa, con sus paredes y habitaciones.

CSS (Presentación)

  • Función: Estiliza la apariencia de la estructura.
  • Pregunta clave: "¿Cómo se ve la página?"
  • Ejemplos: Un párrafo con el texto en color azul y negrita.
  • Metáfora: La decoración de la casa, con la pintura, los muebles y el color de las cortinas.

Trabajan de la mano: primero, usas HTML para construir el andamio, y luego usas CSS para decorarlo. Un buen desarrollador entiende que el contenido y la presentación deben estar separados para mantener un código limpio y eficiente.

Sintaxis básica: selectores, propiedades, valores 
selector {
    propiedad: valor;
    propiedad-2: valor-2;
}
  • Selector: Es la parte que apunta al elemento HTML que quieres estilizar (ej: p, h1, .mi-clase, #mi-id).
  • Propiedad: Es el aspecto del elemento que quieres cambiar (ej: color, font-size, margin-top).
  • Valor: Es el ajuste o la especificación que le das a la propiedad (ej: red, 16px, 10px).

Cada declaración de propiedad-valor termina con un punto y coma (;). El bloque completo de declaraciones va entre llaves ({}).

¿Qué significa que CSS funcione en cascada?

Que CSS funcione en cascada significa que las reglas de estilo no se aplican al azar, sino que siguen un orden de prioridades muy estricto. Cuando un elemento HTML tiene varias reglas de CSS apuntándole desde diferentes lugares (archivos, selectores o incluso en la misma línea de código), el navegador determina cuál de todas es la más "fuerte" y, por lo tanto, la que debe aplicarse.

Este proceso de decisión se basa en tres factores principales que actúan como un sistema de filtrado, de mayor a menor importancia:

  • Especificidad: Esta es la regla más importante. Un selector más específico anula a uno menos específico. Por ejemplo, una regla que usa un ID (#mi-id) es más específica que una que usa una clase (.mi-clase) o una etiqueta (p).
  • Orden del código: Si dos reglas tienen la misma especificidad, la última que aparece en la hoja de estilos es la que gana y se aplica. Por eso, el orden en el que escribes tu código es crucial.
  • Importancia: Las reglas con la propiedad !important tienen la máxima prioridad, incluso por encima de la especificidad. Sin embargo, su uso no es recomendable ya que dificulta la depuración y el mantenimiento del código.
Enlazando HTML y CSS: Tres formas de aplicar estilos

Para que los estilos que definas en tu hoja de CSS afecten a tu documento HTML, necesitas enlazarlos. Existen tres métodos principales para hacerlo, aunque uno es el más recomendado para la mayoría de los casos.

1. Hoja de Estilos Externa (Recomendada)

Es la forma más común y eficiente de trabajar. Los estilos se escriben en un archivo `.css` separado y se enlazan al documento HTML usando la etiqueta `<link>` dentro del `<head>`.

<!-- Enlace en el HTML -->
<head>
    <link rel="stylesheet" href="styles.css">
</head>

<!-- Código en el archivo styles.css -->
body {
    background-color: lightblue;
}

Ventajas: Mantiene la separación de la estructura (HTML) y la presentación (CSS), haciendo tu código más organizado, fácil de mantener y reutilizable en múltiples páginas.

2. Estilos Internos

Los estilos se declaran directamente en el mismo documento HTML, dentro de la etiqueta `<style>` ubicada en la sección `<head>`.

<head>
    <style>
        h1 {
            color: maroon;
        }
    </style>
</head>

Ventajas: Útil para estilos específicos de una sola página o para pruebas rápidas. No requiere un archivo adicional.

3. Estilos en Línea (Inline)

Los estilos se aplican directamente a un elemento HTML usando el atributo `style`.

<h2 style="color: blue; font-size: 20px;">Título azul</h2>

Ventajas: Permite una aplicación de estilo muy rápida y específica a un solo elemento.

---

¿Cuál es la mejor opción y por qué?

La hoja de estilos externa es, sin duda, la mejor opción. La razón principal es la separación de responsabilidades (Separation of Concerns). Al mantener el HTML y el CSS en archivos separados, tu código se vuelve:

  • Más limpio: El HTML solo se encarga de la estructura y el contenido, sin estar "contaminado" con estilos.
  • Más fácil de mantener: Si necesitas cambiar el color de todos los títulos de tu sitio, solo tienes que editar una línea en tu archivo CSS externo, en lugar de ir página por página.
  • Más eficiente: El navegador solo tiene que descargar el archivo CSS una vez, y luego puede almacenarlo en caché para su uso en las demás páginas de tu sitio, lo que mejora la velocidad de carga.

Los estilos internos y en línea son útiles en situaciones muy concretas, como para estilos de correos electrónicos o para pruebas puntuales, pero para la web en general, la hoja de estilos externa es el estándar de la industria y la mejor práctica.

Fundamentos esenciales de CSS 🎨

Una vez que conoces la sintaxis básica, es hora de profundizar en las herramientas que te permiten dar forma y estilo a tus páginas. En esta sección, cubriremos los bloques de construcción más importantes de CSS que usarás en tu día a día como desarrollador.

Selectores: El arte de apuntar a elementos

Los selectores son la forma en que le dices a CSS qué elementos HTML quieres estilizar. Dominarlos es crucial para tener control sobre tu diseño. Aquí están los más comunes:

Selector Sintaxis Ejemplo de Uso
Selector de Elemento p Selecciona todas las etiquetas <p> de la página.
Selector de Clase .mi-clase Selecciona todos los elementos con la clase mi-clase. Es el más flexible y usado.
Selector de ID #mi-id Selecciona el elemento con el ID mi-id. Único por página.
Descendiente article p Selecciona las etiquetas <p> que están dentro de un <article>.
Combinadores div > p Selecciona los <p> que son hijos directos de un <div>.

El uso de selectores de clase te permite reutilizar estilos en múltiples elementos, lo cual es fundamental para un código limpio y mantenible.

Colores y Fondos: Dando vida a tu diseño

Colores

Existen varias maneras de definir colores en CSS:

  • Nombres: Nombres de colores predefinidos (red, blue, green). Fáciles de leer, pero limitados.
  • HEX: Código hexadecimal de 6 dígitos (#FF5733). El estándar de la industria.
  • RGB: Valores de rojo, verde y azul (rgb(255, 87, 51)).
  • HSL: Tono, Saturación y Luminosidad (hsl(10, 100%, 60%)). Intuitivo para diseñadores.
  • RGBA: Igual que RGB, pero con un canal de transparencia (rgba(255, 87, 51, 0.7)). El valor `0.7` crea un 70% de opacidad.

💡 Herramienta Útil: Coolors

Para generar paletas de colores de manera rápida y eficiente, te recomendamos explorar esta herramienta: Coolors. Esta web te permite crear combinaciones de colores armoniosas, exportarlas en diferentes formatos y obtener los códigos **HEX**, **RGB** o **HSL** de forma instantánea. Es un recurso muy valioso para cualquier proyecto de diseño.

Fondos

Las propiedades de fondo te permiten añadir color o imágenes a los elementos:

  • `background-color`: Define un color de fondo sólido.
  • `background-image`: Permite usar una imagen como fondo, generalmente con la función `url()`. Puedes controlar su repetición, posición y tamaño.
  • `background-image` (degradados): Usa las funciones `linear-gradient()` o `radial-gradient()` para crear transiciones de color. Por ejemplo: linear-gradient(to right, red, orange).
Unidades y Tipografía: Midiento y estilizando el texto

Unidades de Medida

Las unidades son clave para definir tamaños y espaciados. Se dividen en absolutas y relativas:

  • Absolutas (fijas): px (píxeles) es la más común. Son útiles cuando necesitas un control exacto.
  • Relativas (flexibles): Son la base del diseño responsivo.
    • %: Relativo al tamaño de su contenedor padre.
    • em: Relativo al tamaño de la fuente de su elemento padre.
    • rem: Relativo al tamaño de la fuente del elemento raíz (`<html>`). ¡Muy recomendable!
    • vw: Relativo al ancho de la ventana del navegador.
    • vh: Relativo al alto de la ventana del navegador.

Tipografía

Puedes controlar el aspecto de tus textos con estas propiedades:

  • `font-family`: El tipo de fuente. Puedes usar fuentes del sistema o de servicios como Google Fonts.
  • `font-size`: El tamaño del texto (ej: 16px, 1.2rem).
  • `font-weight`: El grosor de la fuente (ej: bold, 700).
  • `font-style`: El estilo, como italic.

Para usar Google Fonts, simplemente enlazas la fuente en tu `<head>` y la aplicas con `font-family` en tu CSS.

📐 Maquetación: De principiante a experto en CSS

La maquetación es el corazón de la web visual, es la forma en que organizamos y posicionamos los elementos en la pantalla para crear un diseño coherente. Dominar estos conceptos te permitirá pasar de la teoría a la práctica, construyendo cualquier tipo de diseño que imagines.

El Modelo de Caja: Todo es una caja 📦

En CSS, cada elemento HTML es una caja, y entender cómo funcionan es fundamental. El Modelo de Caja describe cómo el espacio se distribuye alrededor de un elemento.

El modelo de caja se compone de:

  • Contenido (Content): El área central donde se encuentra el texto o la imagen.
  • Relleno (Padding): El espacio transparente entre el contenido y el borde. Es un espacio interno.
  • Borde (Border): La línea que rodea el relleno y el contenido. Puedes darle estilo, color y grosor.
  • Margen (Margin): El espacio transparente fuera del borde. Se usa para separar el elemento de otros elementos adyacentes.
Display y Position: Controlando la naturaleza de los elementos

Las propiedades display y position te permiten controlar cómo se comportan los elementos en el flujo del documento.

display

Define el comportamiento de un elemento. Los valores más comunes son:

  • block: Ocupa todo el ancho disponible, creando un salto de línea antes y después (ej: `div`, `p`).
  • inline: Ocupa solo el espacio necesario para su contenido y no crea saltos de línea (ej: `span`, `a`).
  • inline-block: Combina lo mejor de ambos: se comporta como un elemento en línea pero acepta propiedades de ancho y alto como un bloque.
  • none: Oculta completamente el elemento, sin ocupar espacio.

position

Permite un control más fino sobre la ubicación de un elemento.

  • static: El valor por defecto. El elemento sigue el flujo normal del documento.
  • relative: Se mueve en relación con su posición original, pero sigue ocupando su espacio en el flujo del documento.
  • absolute: Se posiciona en relación a su contenedor posicionado más cercano (no `static`). Se elimina del flujo normal, y otros elementos no dejan espacio para él.
  • fixed: Se posiciona en relación a la ventana del navegador. No se mueve al hacer scroll.
  • sticky: Un híbrido de `relative` y `fixed`. El elemento se comporta como `relative` hasta que el usuario lo desplaza a una posición específica, momento en el que se vuelve `fixed`.
Flexbox y Grid: Maestros de la maquetación

Antes de Flexbox y Grid, la maquetación web era un desafío complejo. Estos dos sistemas modernos te dan un poder sin precedentes para crear cualquier diseño de manera eficiente.

Flexbox: Alineando en una dimensión

Flexbox es ideal para alinear elementos en una sola dimensión, ya sea en una fila o en una columna. Con él, puedes controlar la alineación, el orden y la distribución del espacio entre los elementos de forma sencilla. Es la herramienta perfecta para barras de navegación, menús, o cualquier componente con elementos que necesiten alinearse entre sí.

Para usar Flexbox, simplemente convierte un contenedor en un `flex container` con la propiedad `display: flex;`. A partir de ahí, puedes controlar los elementos hijos con propiedades como `justify-content` y `align-items`.

Grid: Diseñando en dos dimensiones

Si Flexbox es para una dimensión, Grid es para dos. Te permite crear diseños complejos y estructurados con filas y columnas. Es la herramienta definitiva para el diseño de páginas completas, como un blog con múltiples columnas o una galería de imágenes.

Para usar Grid, defines un contenedor como `grid container` con `display: grid;`. Luego, puedes definir explícitamente el tamaño de tus filas y columnas con `grid-template-rows` y `grid-template-columns`.

✨ Estética y experiencia: Dando vida a tu diseño

Una vez que tu maquetación está en su lugar, es momento de darle personalidad a tu web. En este apartado, exploraremos las propiedades que transforman un diseño funcional en una experiencia visual memorable, añadiendo detalles que elevan la calidad y el profesionalismo de tu trabajo.

Bordes, Sombras y Efectos Visuales

Los detalles marcan la diferencia. Con CSS, puedes añadir profundidad y suavidad a tus elementos.

Bordes y esquinas redondeadas

La propiedad `border-radius` es tu aliada para suavizar los bordes de cualquier elemento, dándole un toque moderno y orgánico. 

.caja-suave {
    border-radius: 10px;
}
.circulo {
    border-radius: 50%;
}

Sombras y Transparencias

Las sombras añaden profundidad y ayudan a los elementos a destacar.

  • `box-shadow`: Aplica una sombra a un elemento de caja, como un `div` o una tarjeta. Se controla la posición (x, y), el desenfoque, la dispersión y el color.
  • `text-shadow`: Crea una sombra detrás del texto para un efecto visual dramático.
  • Transparencia: La propiedad `opacity` (de 0.0 a 1.0) hace un elemento más o menos transparente. También puedes usar `rgba()` o `hsla()` para dar transparencia solo al color, manteniendo el elemento sólido.
Movimiento y Dinamismo: Transiciones y Animaciones

Las transiciones y animaciones son clave para crear una experiencia de usuario fluida y reactiva. Hacen que las interacciones se sientan naturales y pulidas.

Transiciones (transition)

Una transición crea un cambio suave entre dos estados de un elemento. Por ejemplo, al pasar el mouse sobre un botón, puedes hacer que su color cambie gradualmente en lugar de hacerlo de golpe. 

.boton {
    background-color: blue;
    transition: background-color 0.3s ease-in-out;
}
.boton:hover {
    background-color: darkblue;
}

La propiedad transition te permite especificar la propiedad a animar, la duración, la función de tiempo (`ease`, `linear`, etc.) y el retraso.

Animaciones (@keyframes)

Las animaciones te dan un control total sobre una secuencia de estados. Usando `@keyframes`, puedes definir puntos de control para que un elemento cambie de estilo de forma progresiva. 

@keyframes vibrar {
    0% { transform: translateX(0); }
    50% { transform: translateX(-5px); }
    100% { transform: translateX(5px); }
}
.icono {
    animation: vibrar 0.5s infinite;
}

Con la propiedad animation, enlazas el elemento a la animación, defines su duración, repetición y otros parámetros.

Pseudo-clases y Pseudo-elementos: Más allá de los selectores básicos

Estas herramientas te permiten seleccionar partes de un elemento o elementos en un estado específico que no puedes seleccionar con selectores de clase o ID.

Pseudo-clases (:)

Las pseudo-clases seleccionan un elemento cuando está en un estado particular.

  • `:hover`: Cuando el cursor está sobre el elemento.
  • `:active`: Cuando se hace clic o se pulsa sobre el elemento.
  • `:focus`: Cuando el elemento recibe el foco del teclado (útil para accesibilidad).
  • `:nth-child(n)`: Selecciona el enésimo elemento hijo de un padre (ej: `li:nth-child(2)` selecciona el segundo elemento de una lista).

Pseudo-elementos (::)

Los pseudo-elementos te permiten estilizar una parte de un elemento.

  • `::before`: Inserta contenido antes del contenido real del elemento. Ideal para iconos o decoraciones.
  • `::after`: Inserta contenido después del contenido real.
  • `::first-letter`: Estiliza la primera letra de un texto.

Dato clave: A menudo se usan `::before` y `::after` junto con la propiedad `content`. Estos pseudo-elementos son muy versátiles y se usan en trucos avanzados de CSS.

📱 Buenas prácticas: Elevando tu CSS a un nivel profesional

Escribir código que funciona es una cosa; escribir código limpio, eficiente y fácil de mantener es otra. En este apartado, cubriremos las prácticas esenciales que todo desarrollador debe dominar para crear estilos de alta calidad, listos para la producción y pensados para el futuro.

Diseño Responsivo: La mentalidad "Mobile First"

El diseño responsivo es la clave para que tu sitio web se adapte perfectamente a cualquier pantalla, desde un teléfono móvil hasta un monitor de escritorio. La técnica principal para lograrlo es el uso de Media Queries. 

/* Media Query: los estilos dentro de este bloque
    solo se aplican cuando el ancho es mayor a 768px */
@media (min-width: 768px) {
    .contenedor {
        display: flex;
    }
}

La filosofía "Mobile First" es la mejor práctica. Esto significa que primero diseñas y estilizas tu sitio para la pantalla más pequeña, y luego usas las Media Queries para añadir estilos que lo mejoren en pantallas más grandes. De esta forma, aseguras la mejor experiencia para la mayoría de los usuarios móviles, y tu código es más eficiente.

Organización y Mantenimiento de tu CSS

Un buen código CSS es un código organizado. Adoptar estas prácticas te ahorrará tiempo y evitará dolores de cabeza a largo plazo.

Reutilización de clases

Evita estilos en línea y repeticiones. Crea clases genéricas para patrones comunes (ej: .card, .button) y reutilízalas en tu HTML. Así, si necesitas cambiar un estilo, solo lo haces en un lugar.

<button class="button primary">Comprar</button>

Comentarios y Variables

Comentar tu código con explicaciones claras hace que sea más fácil de entender, especialmente si trabajas en equipo. Además, el uso de variables CSS (con la sintaxis --nombre-variable) es una herramienta poderosa. 

:root {
  --color-principal: #F16529;
}
p {
  color: var(--color-principal);
}
Accesibilidad y Herramientas de Depuración

Un diseño profesional es un diseño accesible para todos. Tu código debe ser compatible con usuarios de diferentes habilidades y dispositivos.

Accesibilidad en CSS

Presta atención a los contrastes de color para que el texto sea legible (la proporción mínima es 4.5:1). Además, asegúrate de que los elementos interactivos, como enlaces y botones, sean lo suficientemente grandes para ser pulsados fácilmente y que sus estilos de `:focus` sean claros para la navegación con teclado.

Validadores y Herramientas del Navegador

Las herramientas de desarrollador de tu navegador (F12) son tus mejores amigas. Te permiten inspeccionar elementos, ver los estilos aplicados, depurar errores y probar cambios en tiempo real. Utiliza la sección de "Computed Styles" para ver todos los estilos que afectan a un elemento. También puedes usar validadores CSS en línea para revisar la sintaxis de tu código.

Un vistazo al futuro: Preprocesadores CSS

A medida que tus proyectos crezcan, escribir CSS puro puede volverse repetitivo. Los preprocesadores son herramientas que extienden las capacidades de CSS, permitiéndote usar lógica de programación como variables, anidación y funciones.

Sass (Syntactically Awesome Style Sheets) y SCSS (Sassy CSS) son los más populares. Aunque al principio parezcan complejos, son un paso natural para un desarrollador que busca optimizar su flujo de trabajo.

Ejemplo de SCSS: 

$color-principal: #F16529;

.tarjeta-producto {
  background-color: $color-principal;
  .titulo {
    font-size: 2rem;
  }
}

Este código se compila automáticamente a CSS, permitiéndote escribir estilos de una manera más potente y organizada.

Errores comunes que debes evitar ⚠️

Conocer las malas prácticas es tan importante como dominar las buenas. Aquí hay un par de errores que debes evitar para mantener tu código profesional:

Abuso de !important

Esta declaración fuerza una propiedad a ser la más importante, ignorando la cascada y haciendo tu código difícil de depurar. Úsalo solo como último recurso y con extrema precaución.

Estilos en línea

Como ya vimos, usar el atributo style directamente en el HTML rompe la separación de responsabilidades, haciendo que tu código sea desordenado y difícil de mantener.

Al evitar estas prácticas, no solo escribirás un CSS más eficiente, sino que también te prepararás para colaborar en proyectos más grandes y complejos.

🚀 Extra: Introducción a Bootstrap

Ya que has dominado los fundamentos de HTML y CSS, es el momento de dar el siguiente paso y aprender a trabajar con un framework de CSS. Un framework es como una caja de herramientas prefabricada, llena de componentes y reglas que te permiten construir diseños de forma mucho más rápida y eficiente. El más popular de todos es Bootstrap.

¿Qué es Bootstrap y cómo funciona?

Bootstrap es un framework de código abierto para el desarrollo web. Ofrece una colección de plantillas de diseño basadas en HTML y CSS, así como componentes prefabricados de JavaScript. Su principal objetivo es ayudarte a crear sitios web y aplicaciones web responsivas y profesionales sin tener que escribir cada línea de CSS desde cero.

En esencia, es una biblioteca de clases CSS que puedes aplicar a tus elementos HTML. Por ejemplo, en lugar de escribir estilos para un botón, simplemente añades la clase btn btn-primary a la etiqueta <button>. Bootstrap ya ha definido cómo se verá y se comportará ese botón.

Ventajas clave

  • Rapidez de desarrollo: Con componentes listos para usar como menús de navegación, tarjetas, formularios y botones, puedes maquetar un sitio completo en una fracción del tiempo que te llevaría con CSS puro.
  • Diseño responsivo: Bootstrap se basa en un sistema de rejilla que facilita la creación de diseños que se ven bien en cualquier dispositivo.
  • Consistencia: Todos los componentes de tu sitio mantendrán una apariencia coherente y profesional.
  • Comunidad y soporte: Cuenta con una enorme comunidad, extensa documentación y una gran cantidad de recursos.

¿Cómo se usa?

Para empezar, solo necesitas enlazar su hoja de estilos y sus scripts de JavaScript en tu documento HTML, de forma similar a como lo harías con un archivo CSS externo. 

<!-- Enlace en el head -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">

<!-- Enlace al final del body -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
El sistema de rejilla y un ejemplo práctico

El principal concepto de Bootstrap es su sistema de rejilla. Usando las clases container, row y col, puedes dividir la página en columnas, controlando cómo se distribuyen en diferentes tamaños de pantalla. Esto simplifica enormemente la creación de diseños complejos y responsivos.

Aquí tienes un ejemplo de un componente de tarjeta que puedes crear con unas pocas clases de Bootstrap. 

<div class="card" style="width: 18rem;">
  <img src="..." class="card-img-top" alt="...">
  <div class="card-body">
    <h5 class="card-title">Título de la tarjeta</h5>
    <p class="card-text">Aquí va el contenido de la tarjeta.</p>
    <a href="#" class="btn btn-primary">Ir a algún lado</a>
  </div>
</div>
El componente Carousel de Bootstrap

El Carousel de Bootstrap es un componente de presentación de diapositivas que permite mostrar imágenes, texto o contenido personalizado en un formato de carrusel rotatorio. Es completamente funcional y responsivo, lo que lo hace ideal para crear galerías de imágenes o banners deslizantes en tu sitio web.

Para usarlo, necesitas un contenedor principal con la clase .carousel y otros elementos para las diapositivas (.carousel-item) y los controles de navegación (flechas) e indicadores (puntos). El atributo data-bs-slide es clave para su funcionamiento.

¿Por qué aprender Bootstrap después de CSS puro?

Aunque Bootstrap es increíblemente útil, es crucial que primero domines CSS puro. Aprender los fundamentos te da la comprensión necesaria de lo que está sucediendo "detrás de las escenas". Sin ese conocimiento, serías un simple "usuario" de un framework, incapaz de depurar o personalizar algo que no encaje en el diseño. Dominar CSS es lo que te permite manipular y extender Bootstrap, no solo usarlo por defecto, lo que te convierte en un desarrollador más completo y competente.
