Creando Formularios con HTML

Los formularios son la puerta de entrada a la interactividad en la web. Son los elementos que permiten a los usuarios enviar datos a un servidor, ya sea para iniciar sesión, enviar un mensaje o realizar una compra. Dominar su estructura y validación es un paso fundamental para cualquier desarrollador web.

En esta sección, exploraremos cómo construir formularios robustos y accesibles utilizando las etiquetas y atributos nativos de HTML. Aprenderás a usar diferentes tipos de campos, a agrupar información de manera semántica y a implementar validaciones para mejorar la experiencia del usuario sin necesidad de JavaScript.

📝 Introducción a los Formularios

Los formularios son la columna vertebral de la interacción en la web. Nos permiten capturar información del usuario, lo que a su vez impulsa aplicaciones dinámicas.

Estructura Básica: `form`, `fieldset` y `label`

Para construir un formulario, necesitas tres etiquetas principales que le dan estructura y semántica.

<form>: El Contenedor

Es la etiqueta que envuelve todos los campos. Es un contenedor que define dónde y cómo se enviarán los datos (a través de los atributos action y method).

<form action="/submit-form" method="POST">
  ... campos del formulario ...
</form>

<fieldset> y <legend>: Agrupando campos

<fieldset> agrupa campos relacionados y <legend> le da un título a ese grupo. Esto mejora la accesibilidad y la organización visual.

<fieldset>
  <legend>Datos de envío</legend>
  <!-- Campos relacionados aquí -->
</fieldset>

<label>: La clave de la accesibilidad

La etiqueta <label> asocia una descripción con un campo de formulario. El atributo for del <label> debe coincidir con el id del campo. Esto permite que el usuario haga clic en la etiqueta para activar el campo, lo que es vital para la usabilidad y la accesibilidad.

<label for="nombre">Tu nombre</label>
<input type="text" id="nombre">
Tipos de Input: Elige el campo correcto

HTML5 nos ofrece una amplia variedad de tipos de `input` que no solo cambian la apariencia, sino que también mejoran la experiencia de usuario y la validación automática.

Inputs de texto y números

  • `text`: Campo de texto de una sola línea.
  • `email`: Campo para correos electrónicos. Valida que el formato sea correcto.
  • `password`: Campo para contraseñas, oculta los caracteres.
  • `number`: Campo para números, permite usar flechas para aumentar o disminuir.
  • `tel`: Campo para números de teléfono. Mejora la usabilidad en móviles.
  • `textarea`: Área de texto de varias líneas.

Inputs de selección y botones

  • `radio`: Para elegir una única opción de un grupo.
  • `checkbox`: Para elegir una o varias opciones.
  • `select` y `option`: Un menú desplegable de opciones.
  • `button`: Un botón genérico.
  • `submit`: Un botón para enviar el formulario.
  • `reset`: Un botón para limpiar todos los campos del formulario.
Validación de HTML: Atributos sin JavaScript

Con HTML5, puedes validar campos directamente en el navegador usando atributos. Esto ofrece una capa de validación básica antes de que los datos lleguen al servidor.

Atributos esenciales

  • `required`: Hace que un campo sea obligatorio. El formulario no se enviará si está vacío.
  • `minlength` / `maxlength`: Define el número mínimo y máximo de caracteres.
  • `min` / `max`: Para inputs de tipo `number`, define el rango de valores permitidos.
  • `placeholder`: Muestra un texto de ejemplo dentro del campo.
  • `pattern`: Usa una expresión regular para validar el formato del campo (ej: para números de teléfono o códigos postales).

Ejemplo de uso

<label for="nombre">Nombre</label>
<input type="text" id="nombre" required minlength="3">

<label for="email">Email</label>
<input type="email" id="email" required>

✍️ Ejercicio: Construye un Formulario de Contacto

Combina todo lo aprendido para construir un formulario de contacto completo.

Objetivo

  • Crea un formulario con campos para nombre, email, teléfono, asunto (un `select`) y mensaje (un `textarea`).
  • Usa `<fieldset>` y `<legend>` para agrupar la información.
  • Asegúrate de que el nombre, email, asunto y mensaje sean campos obligatorios (`required`).
  • Usa `minlength` para el campo de nombre.
  • Añade un `pattern` para el campo de teléfono, validando que sean 9 dígitos numéricos.
  • Incluye un `checkbox` para la aceptación de términos y condiciones que también sea `required`.
  • Añade un botón de envío (`submit`) y otro para limpiar el formulario (`reset`).

Con este ejercicio, tendrás un formulario funcional y validado, demostrando tu comprensión de los fundamentos de HTML para la interacción.
